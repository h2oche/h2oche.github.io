a == b && b == c && c != a 인 a,b,c의 자료형과 값을 찾는 문제이다.
컴퓨터가 어떻게 실수를 처리하는지, 형변환이 어떻게 일어나는지를 알아야 풀 수 있는 문제다.

컴퓨터는 부동 소수점으로 실수를 표현하고 그 정확성에 따라 float, double, long double 등으로 나뉜다.
또 int -> long long -> float -> double 의 priority를 가지고 형변환을 하게 되고, 연산을 할 때 자료형이 다르면 낮은 priority의 값을 높은 priority의 값으로 형변환 시킨다.
예를 들어 (int)10 == (float)11.1 을 계산할때 10이 10.0으로 바뀐다음 == 을 계산한다는 의미이다.

0, 1로 모든 실수들을 표현할 수 없으므로, 당연히 오차가 발생하게 된다. 부동 소수점 방식은 부호, 지수, 가수 부분으로 나뉜다.
float(32) : 부호 1bit + 지수 8bit + 가수 23bit
double(64) : 부호 1bit + 지수 11bit + 가수 52bit
자세한 설명은 링크를 보면 된다.

즉, 정수 1111…1(1이 24개)은 float으로 변환 될 때, 가수가 23bit 이므로 마지막 값이 떨어져 나가서 오차가 발생하게 된다.
