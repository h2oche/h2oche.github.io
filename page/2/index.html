작성일 2019-01-23 | In algorithm , boj

a == b && b == c && c != a 인 a,b,c의 자료형과 값을 찾는 문제이다.
컴퓨터가 어떻게 실수를 처리하는지, 형변환이 어떻게 일어나는지를 알아야 풀 수 있는 문제다.

컴퓨터는 부동 소수점으로 실수를 표현하고 그 정확성에 따라 float, double, long double 등으로 나뉜다.
또 int -> long long -> float -> double 의 priority를 가지고 형변환을 하게 되고, 연산을 할 때 자료형이 다르면 낮은 priority의 값을 높은 priority의 값으로 형변환 시킨다.
예를 들어 (int)10 == (float)11.1 을 계산할때 10이 10.0으로 바뀐다음 == 을 계산한다는 의미이다.

0, 1로 모든 실수들을 표현할 수 없으므로, 당연히 오차가 발생하게 된다. 부동 소수점 방식은 부호, 지수, 가수 부분으로 나뉜다.
float(32) : 부호 1bit + 지수 8bit + 가수 23bit
double(64) : 부호 1bit + 지수 11bit + 가수 52bit
자세한 설명은 링크를 보면 된다.

즉, 정수 1111…1(1이 24개)은 float으로 변환 될 때, 가수가 23bit 이므로 마지막 값이 떨어져 나가서 오차가 발생하게 된다.

boj15549

작성일 2019-01-23 | In algorithm , boj

x != 0 && x == -x 인 x의 자료형과 값을 추측하는 문제이다. 처음 봤을 때 무슨 소리인가 하고 생각했는데,
컴퓨터에서 음수가 어떻게 표현이 될까 생각해 보다가 답을 찾을 수 있었다.

컴퓨터는 정수에 한해서 2의 보수 형태로 음수를 만들어 낸다. 2의 보수를 만드는 방법은 모든 비트를 반전시킨 후, 1을 더하면 된다.
예를 들어 4bit 짜리 정수 자료형을 생각해보면, 0010 -> 1101 -> 1110 이 되어서 0010은 2를 나타내고 1110은 -2를 표현하게 된다.
이렇게 되면 x + (-x) = 0이 자연스럽게 계산된다는 특징이 있다.

그런데 만약 1000의 2의 보수를 생각해 보면, 1000 -> 0111 -> 1000 이 되어서 원래의 값과 똑같게 됨을 알 수 있다.
즉, 첫번째 bit만 1이고 나머지는 0이면 어떤 정수 자료형이든 x == -x 가 된다.

처음에 int 0x80000000으로 제출했는데, 0x를 인식못하는 거 같아서 10진수 표현으로 바꿨더니 통과했다.

boj9251

작성일 2019-01-22 | In algorithm , boj
  • 시간복잡도 : O(NM)
  • 점화식
    • a[i] == b[j] 인 경우
      • dp[i][j] = dp[i-1][j-1] + 1
    • a[i] != b[j] 인 경우
      • max(dp[i][j-1], dp[i-1][j])

Longest Common Subsequence(LCS)의 길이를 구하는 문제이다.
LCS는 string A,B의 subsequece중 가장 긴 것을 의미한다(ex: LCS(“ABCDE”, “BXCXE”) = “BCE”, LCS(“BA”, “AB”) = {“A”, “B”})
dp[i][j]를 a[0…i], b[0…j] 의 LCS의 길이라고 정의하고, 다음과 같이 점화식을 세운 후, dp를 채우면 된다.

  • a[i] == b[j] 인 경우
    • dp[i][j] = dp[i-1][j-1] + 1
  • a[i] != b[j] 인 경우
    • max(dp[i][j-1], dp[i-1][j])

대충 원리를 생각해 보면 Ax, By 가 있을 때 x == y 이면 LCS(Ax,By) = LCS(A,B) + 1 이고, 아닌 경우 LCS(Ax,By) = max(LCS(A,By), LCS(Ax,B)) 이므로 위와 같은 점화식이 나온다.

LCS는 굉장히 다양한 곳에서 응용이 가능한 알고리즘이다. Edit Distacne와 Diff를 구할 때도 쓰인다.
이 방법 말고도 greedy한 방법으로 더 빠르게 LCS를 구하는 알고리즘이 있는데, 자세한 것은 An O(ND) Difference Algorithm and Its Variations 이라는 제목의 논문 보면 될 것 같다.

응용 버전으로 9252번 문제가 있다.

작성일 2019-01-22 | In algorithm , boj
  • 시간복잡도 : O(N+K)

싸이클이 없는 것이 보장되었기 때문에 함수를 활용한 dp로 비교적 쉽게 풀 수 있었다.
transform, max_element 같은 STL 함수를 써보았다.
함수를 활용했더니 시간복잡도 분석이 한 눈에 들어오지는 않는데, 아마 대충 O(N+K) 정도 될거같은데, K가 훨씬 크므로 O(K)로 보아도 무방할 것 같다.
dp[i]가 의미하는 것은 건물 i 를 짓는데 필요한 최소 시간이다.
건물 i 는 하위 테크 건물들이 걸리는 시간 중 최댓값과 건물 i 자신을 짓는데 걸리는 시간의 합으로 결정된다.

boj11066

작성일 2019-01-22 | In algorithm , boj
  • 시간복잡도 : O(K^3)
  • 점화식
    • dp[i][j][1] = min{dp[m][j][0] + dp[m][j][1] + dp[i-m][j+m][0] + dp[i-m][j+m][1] | m = 1, …, i-1}
    • dp[i][j][0] = dp[m][j][0] + dp[i-m][j+m]0

핵심 아이디어는 파일을 합치는데 드는 비용과 그 파일을 만드는 데 까지 필요한 비용, 이렇게 2가지를 계속 유지하는 것이다.

dp의 의미는 다음과 같다.
dp[i][j] : j번째 파일부터 j+i번째 파일까지 합쳐서 임시파일을 만들 때 드는 최소비용 2가지를 저장함
dp[i][j][0] : 이 임시파일을 나중에 다른 파일과 합칠 때 드는 비용
dp[i][j][1] : 이 임시파일이 만들어지기까지 드는 최소비용

점화식은 다음과 같다.

  • dp[i][j][1] = min{dp[m][j][0] + dp[m][j][1] + dp[i-m][j+m][0] + dp[i-m][j+m][1] | m = 1, …, i-1}
  • dp[i][j][0] = dp[m][j][0] + dp[i-m][j+m]0
