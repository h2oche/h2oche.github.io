  • 시간복잡도 : O(N)
  • 점화식 : dp[j][k | MASK(j)] = dp[j+1, j-1][k]

bitmask와 다이나믹 프로그래밍을 적절히 활용해서 푼 문제이다.
dp[i][k]의 의미는 끝자리가 i로 끝나면서 k에 표시된 수들을 사용한 계단수이다.
N을 차례대로 늘려가면서 dp를 갱신하면 dp[i][1023]에 i로 끝나면서 모든 자릿수가 포함된 계단수의 갯수를 얻을 수 있다.
점화식은 dp[j][k | MASK(j)] = dp[j+1, j-1][k] 이다.

boj2098

작성일 2019-01-21 | In algorithm , boj
  • 시간복잡도 : O(N * 2^N)

bitmask와 함수를 활용한 다이나믹 프로그래밍을 적절히 활용해서 푼 문제이다.
dp[i][mask]의 의미는 0번 지점부터 i번 지점까지 mask에 포함된 점들을 통해서 가는 경로 중 최솟값이다.
get_shortest(j, mask)는 0번 부터 j번 지점까지 mask에 포함된 점들을 통해서 가는 경로 중 최솟값을 반환한다.
시간복잡도가 O(N * 2^N) 이기 때문에(실제로는 이것보단 빠를 것 같다) N이 매우 작을 때만 가능한 풀이이다.

boj11062

작성일 2019-01-21 | In algorithm , boj
  • 시간복잡도 : O(N^2)
  • 점화식
    • j + i 번째 카드를 선택할 경우
      • dp[i][j][0] = dp[i-1][j][1] + card[j+i]
      • dp[i][j][1] = dp[i-1][j][0]
    • j 번째 카드를 선택할 경우
      • dp[i][j][0] = dp[i-1][j+1][1] + card[j]
      • dp[i][j][1] = dp[i-1][j][0]

다이나믹 프로그래밍을 활용해 푼 문제이다.
dp[i][j][0]과 dp[i][j][1]의 의미는 각각 j번째 카드에서부터 j+i번째 카드까지가 주어졌을 때, 첫 번째 사람이 고를 수 있는 최댓값, 두 번째 사람이 고를 수 있는 최댓값을 의미한다.
i가 늘어날 때, j + i번째 카드를 먼저 선택할 것인지, j번째 카드를 먼저 선택할 것인지로 경우가 나뉘고, 이 때 최댓값을 선택하면서 dp를 갱신하면 된다.

점화식은 다음과 같다.

  • j + i 번째 카드를 선택할 경우
    • dp[i][j][0] = dp[i-1][j][1] + card[j+i]
    • dp[i][j][1] = dp[i-1][j][0]
  • j 번째 카드를 선택할 경우
    • dp[i][j][0] = dp[i-1][j+1][1] + card[j]
    • dp[i][j][1] = dp[i-1][j][0]

글을 쓰다보니 dp의 첫번째 인덱스는 없앨 수 있을 것 같다. 부분합을 구해놓는다면, 마지막 인덱스도 굳이 필요없을 것 같다(부분합[i][j] = dp[i][j][0] + dp[i][j][1] 이므로).
하지만 이렇게 해도 넉넉히 통과하기 떄문에 더 이상 수정하지 않았다.
